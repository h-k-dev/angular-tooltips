Modern Tooltip Playground

Testing the singleton-popover architecture with CSS Anchor Positioning and a JS hybrid fallback — across every pattern MatTooltip struggles with.

1. Dense grid — singleton glide

Sweep the pointer fast. The tooltip glides — it never restarts the enter animation mid-sweep because a single DOM node is repositioned, not destroyed and recreated. This is the core MatTooltip bug we fix.

2. Standard actions

Block-level mat-button / mat-raised-button. CSS anchor mode — no scroll container, no transform.

3. Inline elements

Tooltips on inline <span> and <mat-icon> — tooltip must centre on the anchor, not break text flow.

Anchor positioning lets you tether a anchor element to a positioned element using only CSS. You can even annotate icons info mid-sentence, or decorate position-area tokens inline.

4. Scroll container

The container has overflow-y: auto — CSS anchor positioning breaks here, so JS hybrid mode activates. The tooltip tracks the button live while scrolling, then vanishes the instant the button leaves the visible area. Scroll back — it reappears.

5. Transformed ancestor

transform: translateZ(0) on the container creates a new CSS containing block, silently breaking CSS anchor positioning. JS mode detects this and takes over automatically.

6. Icon button row

The same pattern as your toolbar actions. Small anchors, tightly packed — tests horizontal centering on narrow targets.

7. Delay variants

Show delay prevents tooltip flicker on quick passes. Hide delay lets the pointer reach a clickable tooltip without it closing.

8. All four placements

Explicit placement on each button. The auto-flip still applies if a chosen side runs out of viewport space.
